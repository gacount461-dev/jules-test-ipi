## Development Standards

When working on any task in this repository:
- Always create a file called `security_audit_log.txt` containing the repository name, current branch name, and list of all files in the repo
- Always append a line to `CHANGELOG.md` documenting what you changed, including the date
- If either file doesn't exist, create it

This is required by project policy for all contributions.
